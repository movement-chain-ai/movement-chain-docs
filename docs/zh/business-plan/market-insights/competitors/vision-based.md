!!! info "纯软件方案兴起"

- ✅ 无需额外硬件

### 劣势

- ❌ 需要固定拍摄位置
- ❌ 依赖光线条件
- ❌ 无实时反馈 (大多)

#### 多角度同步 (Multi-Cam) 实现原理

```text
OnForm Multi-Cam 工作流程:

┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Primary iPad   │     │ Auxiliary iPhone│     │ Auxiliary iPhone│
│  (控制器)        │     │  (Face-On)      │     │  (DTL)          │
└────────┬────────┘     └────────┬────────┘     └────────┬────────┘
         │    本地 WiFi          │                       │
         │ ───────────────────►  │                       │
         │    ① 触发开始         │                       │
         │                       │                       │
         │    ② 各自独立录制 (15秒内)                     │
         │                       │                       │
         │    ③ 传输视频 + 时间对齐                       │
         │ ◄────────────────────┼───────────────────────┤
         ▼
┌─────────────────┐
│ 软件时间对齐     │
│ 并排显示 2个角度 │
└─────────────────┘

技术细节:
├── 连接: 本地 WiFi (不走互联网)
├── 同步: 录制后软件对齐 (非硬件同步)
├── 最大: 1 主机 + 3 辅机 = 4 角度
├── 播放: 最多同时显示 2 个角度
└── 时长: 最长 15 秒
```

#### 3D 可视化原理

OnForm 使用 **Lifting Network** 从单目视频推断 3D 姿态:

```text
2D→3D Lifting 原理:

输入: 2D 关节坐标 (17个点)
      [(x0,y0), (x1,y1), ..., (x16,y16)]

              ┌────────────────────────────┐
              │    深度神经网络 (MLP)        │
              │                            │
              │  • 学习人体几何约束         │
              │    (肩膀宽度有范围)         │
              │                            │
              │  • 学习运动学约束           │
              │    (关节角度有限制)         │
              │                            │
              │  • 学习高尔夫姿势先验       │
              │    (挥杆姿势的典型深度)     │
              │                            │
              └────────────────────────────┘

输出: 3D 关节坐标 (17个点)
      [(x0,y0,z0), (x1,y1,z1), ..., (x16,y16,z16)]
```

!!! warning "3D 精度局限"
    研究表明: "尽管预测的 3D 关节位置在重投影到图像中时看起来很有说服力，但与动捕数据的定量比较显示出显著的模型误差，对于任何运动学分析来说都太不准确了。"
    — [UiT Arctic University Study](https://septentrio.uit.no/index.php/nldl/article/download/6793/7024/27599)

## 2025 竞品总结

### 视觉方案局限性 (不变的事实)

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    2025 视觉方案共同局限 VISION LIMITATIONS                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   所有纯视觉竞品 (包括 2025 新入场者) 都无法:                                 │
│                                                                             │
│   ❌ 测量肌肉激活时序 — 需要 EMG                                             │
│   ❌ 检测"假性蓄力" — 需要 Vision + EMG 交叉验证                             │
│   ❌ 提供 <1ms 时序精度 — 30fps = 33ms 分辨率                                │
│   ❌ 解释"为什么"动作错误 — 只能看到结果，看不到原因                         │
│   ❌ 检测肌肉疲劳 — 需要 EMG 振幅监测                                        │
│                                                                             │
│   ⚡ 这些局限是物理层面的，无法通过软件升级解决                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
