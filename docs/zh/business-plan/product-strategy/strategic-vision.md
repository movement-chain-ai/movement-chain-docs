# 战略愿景

> **阅读时间**: ~10 分钟 | **前置知识**: [产品战略](product-strategy.md) | **目标**: 理解公司长期定位与 MVP 验证逻辑

---

## 1. 公司定位

!!! tip "核心定位"
    **Movement Chain AI 是一家 Sports Tech（智能运动）公司，致力于打造完整的 AI 运动服务生态。**

### 1.1 我们不是硬件公司

| 维度 | 硬件公司 | Movement Chain AI |
|-----|---------|-------------------|
| **收入模式** | 一次性销售 | 持续订阅 + 生态分成 |
| **用户关系** | 买完即止 | 长期陪伴 |
| **护城河** | 供应链、制造 | 数据 + 算法 + 用户习惯 |
| **扩展性** | 新品研发周期长 | 接入第三方设备快 |
| **核心资产** | 库存、模具 | 用户数据、AI 模型 |

### 1.2 核心能力

```text
┌─────────────────────────────────────────────────────────────────┐
│                    AI 运动服务生态                                │
├─────────────────────────────────────────────────────────────────┤
│  服务层    │ 个性化训练计划 │ AI 教练反馈 │ 社区 │ 内容               │
├─────────────────────────────────────────────────────────────────┤
│  AI 引擎   │ 多模态融合 │ 运动分析 │ 个性化推荐 │ 进步追踪             │
├─────────────────────────────────────────────────────────────────┤
│  数据层    │ Vision │ IMU │ EMG │ 心率 │ 足压 │ 睡眠 │ ...         │
├─────────────────────────────────────────────────────────────────┤
│  硬件层    │ 自研传感器 │ 第三方可穿戴设备 │ 手机摄像头                │
└─────────────────────────────────────────────────────────────────┘
```

- **数据融合**: EMG 肌肉数据 + 多模态传感器对齐
- **AI 分析**: 运动生物力学分析 + 个性化反馈生成
- **服务闭环**: 从数据采集到行为改变的完整链路

### 1.3 长期愿景：打造完整运动生态

!!! tip "生态定位"
    **用户可以完全依赖 Movement Chain AI 实现自己的运动目标**

    不是"工具"，而是"全程陪伴的 AI 教练"

### 1.4 市场切入策略

```text
Phase 1: 高尔夫 (验证平台)
    ↓ EMG 价值验证成功
Phase 2: 网球/棒球 (扩展挥拍类运动)
    ↓
Phase 3: 健身/力量训练 (核心目标市场)
    ↓
Phase 4: 完整运动生态 (待探索)
```

## 2. 为什么高尔夫是第一步 {#为什么高尔夫是第一步}

| 维度 | 优势 |
|------|------|
| **技术验证** | 动作标准化程度高，EMG 价值可量化 |
| **商业验证** | 用户付费意愿强，客单价高 |
| **团队练兵** | 3人团队跑通硬件+软件+算法全流程 |
| **复用价值** | 时间同步、EMG算法、AI反馈框架可直接复用 |
| **竞品验证** | HackMotion、Sportsbox 已证明市场存在 |

### 2.1 MVP 双重目标 {#mvp-双重目标}

| 目标 | 验证内容 | 成功标准 |
|------|---------|---------|
| **产品验证** | 用户愿意为 Vision + IMU + AI 反馈付费吗？ | 付费转化率 >10% |
| **EMG 验证** | EMG 能带来足够额外价值让用户付更多钱吗？ | Pro 版占比 >20% |

### 2.2 EMG 验证的战略意义 {#emg-验证的战略意义}

**EMG 在消费级运动市场有商业价值**

#### 2.2.1 运动类型分类：发力驱动 vs 动作驱动

运动科学中，不同运动对肌肉激活检测的需求程度差异巨大：

| 类型 | 定义 | 判断标准 | EMG 必要性 |
|------|------|----------|-----------|
| **发力驱动型** | 动作正确 ≠ 发力正确 | 同样的外观轨迹，不同肌群可完成 | 必须 |
| **动作驱动型** | 动作正确 → 发力自然正确 | 动作轨迹决定发力模式 | 不需要 |
| **混合型** | 部分阶段动作驱动，部分阶段发力驱动 | 需具体分析 | 部分需要 |

##### 发力驱动型运动 (EMG 必须)

```text
核心特征:
├── 多条肌肉可以完成同一个动作
├── 外观相同但效率/安全性差异大
├── "代偿"问题普遍存在
└── Vision + IMU 完全无法区分
```

| 运动大类 | 具体项目 | 典型问题 |
|---------|---------|---------|
| **力量训练** | 深蹲、硬拉、卧推、引体向上 | 目标肌群 vs 代偿肌群 |
| **康复训练** | 术后康复、肌肉激活 | 萎缩肌群是否真的在工作 |
| **格斗/武术** | 拳击、柔道、摔跤、跆拳道 | 发力链: 腿→髋→核心→手臂 |
| **等长收缩** | 平板支撑、靠墙静蹲、瑜伽保持 | 看不见运动 = 无法分析 |
| **划船/自行车** | 赛艇、皮划艇、场地自行车 | 左右不平衡、疲劳代偿 |

##### 动作驱动型运动 (Vision + IMU 足够)

```text
核心特征:
├── 动作轨迹高度标准化
├── 外部评判（裁判/计时）为主
├── 动作对了，发力自然不会错
└── 问题出在"姿态"而非"哪块肌肉"
```

| 运动大类 | 具体项目 | 原因 |
|---------|---------|------|
| **技巧评分类** | 跳水、花样游泳、艺术体操 | 评分看外观，动作即结果 |
| **纯耐力类** | 马拉松、竞走、铁人三项 | 心肺/配速主导，非力量主导 |
| **器械操控类** | 帆船、马术、射击(稳定性) | 人机配合 > 肌肉发力 |
| **团队战术类** | 足球、篮球、排球 | 战术跑位 > 个人发力 |

##### 混合型运动 (部分阶段需要 EMG)

| 运动 | 动作驱动阶段 | 发力驱动阶段 |
|------|-------------|-------------|
| **高尔夫** | 站位、握杆、上杆幅度 | 下杆发力顺序、运动链 |
| **网球/羽毛球** | 拍面角度、步法移动 | 挥拍发力链、转体顺序 |
| **田径投掷** | 投掷角度、出手点 | 腿→髋→躯干→手臂的顺序 |
| **游泳** | 划水轨迹、转身技术 | 肩袖激活、核心稳定 |
| **举重** | 杠铃轨迹、接杠位置 | 腿蹬 vs 腰拉 vs 肩拉 |

!!! tip "核心洞察"
    **发力驱动 = "用什么肌肉"比"做什么动作"更重要**

    **动作驱动 = "做什么动作"决定了"用什么肌肉"**

    EMG 的价值 = 解决"发力驱动型"运动中 Vision + IMU 的盲区

### 2.3 验证方法

| 维度 | 验证指标 | 成功标准 |
|------|---------|---------|
| **穿戴接受度** | 试用 10 次后继续使用率 | >60% |
| **感知价值** | "肌肉反馈有用" 评分 | >4/5 |
| **付费意愿** | Pro 版转化率 | >20% |
| **行为改变** | 有 EMG 组 vs 无 EMG 组进步速度 | 显著差异 |

### 2.4 验证成功的意义

- 证明消费者愿意为肌肉数据付费
- 可复制到多运动场景的核心技术资产
- 健身市场扩展有数据支撑
- 融资故事更有说服力

### 2.5 失败预案 (Plan B) {#失败预案}

| EMG 验证失败原因 | 应对策略 |
|-----------------|---------|
| 穿戴太麻烦 | 改进硬件形态（更轻、织物电极） |
| 数据看不懂 | 不是 EMG 问题，改进 AI 解读 |
| 没有感知价值 | 放弃 EMG，专注 Vision + IMU + AI |
| 不愿付费 | EMG 作为免费功能吸引用户，靠其他功能变现 |

## 3. 扩展路径候选 {#扩展路径候选}

基于前文的运动类型分类，按 EMG 战略价值排序：

### 3.1 第一梯队：发力驱动型（EMG 最高价值）

| 运动 | 运动类型 | EMG 核心价值 | 市场规模 | 技术复用度 | 优先级 |
|------|---------|-------------|---------|-----------|--------|
| **健身/力量训练** | 发力驱动 | 代偿检测、目标肌群激活 | 最大 (全球$960亿) | 高 (EMG算法直接复用) | ⭐ **核心目标** |
| **康复训练** | 发力驱动 | 萎缩肌群激活验证 | 大 | 高 | 备选1 |
| **格斗/武术** | 发力驱动 | 发力链顺序、出拳效率 | 中 | 中 | 备选2 |

### 3.2 第二梯队：混合型（EMG 中高价值）

| 运动 | 运动类型 | EMG 核心价值 | 市场规模 | 技术复用度 | 优先级 |
|------|---------|-------------|---------|-----------|--------|
| **棒球** | 混合型 | 挥棒发力链（≈高尔夫） | 大（美国） | 最高 (动作相似) | 备选3 |
| **网球/球拍类** | 混合型 | 挥拍发力链、转体顺序 | 大 | 高 | 备选4 |
| **赛艇/划船** | 混合型 | 左右平衡、腿-背-臂顺序 | 中 | 中 | 备选5 |
| **举重** | 混合型 | 腿蹬 vs 腰拉区分 | 中 | 中 | 备选6 |

### 3.3 第三梯队：动作驱动型（EMG 价值有限）

| 运动 | 运动类型 | 说明 | 建议 |
|------|---------|------|------|
| **跑步** | 混合型 | 臀肌 vs 小腿有价值，但市场竞争激烈 | 低优先级 |
| **体操/跳水** | 动作驱动 | 动作即结果，EMG 价值低 | 不推荐 |
| **团队球类** | 动作驱动 | 战术 > 发力 | 不推荐 |

!!! tip "扩展策略"
    **Phase 2: 网球/棒球** — 优先扩展挥拍类运动

    - 动作模式与高尔夫相似度最高（挥拍发力链）
    - 技术复用度最高（IMU+EMG 算法直接复用）
    - 用户画像重叠（高尔夫 ↔ 网球 ↔ 棒球用户群体交叉）

    **Phase 3: 健身/力量训练** — 核心目标市场

    - EMG 价值最高（代偿检测是刚需）
    - 市场规模最大（全球健身房用户 2 亿+）
    - 用户教育成本低（健身用户理解"肌肉激活"）

## 4. 完整运动生态 {#phase-4-完整运动生态}

### 4.1 核心理念：用户目标达成闭环

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Movement Chain AI 服务闭环生态                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │                         用户目标 (Goal)                              │   │

│                                   │                                         │

│   │  1️⃣ 评估 (Assess)                                                   │   │
│   │  ├── 初始能力测试 (挥杆分析、力量测试、体态评估)                          │   │
│   │  ├── 弱点识别 (Vision+IMU+EMG 多维诊断)                               │   │
│   │  └── 个性化基线建立                                                   │   │
│   └───────────────────────────────┬─────────────────────────────────────┘   │

│   └───────────────────────────────┬─────────────────────────────────────┘   │
│                                   │                                         │
│   ┌───────────────────────────────▼─────────────────────────────────────┐   │
│   │  3️⃣ 训练 (Train)                                                    │   │
│   │  ├── 实时动作指导 (语音/视觉/触觉反馈)                                  │   │
│   │  ├── 每次练习即时纠正                                                  │   │
│   │  └── 训练内容推荐 (练习、视频、文章)                                    │   │
│   └───────────────────────────────┬─────────────────────────────────────┘   │
│                                   │                                         │
│   ┌───────────────────────────────▼─────────────────────────────────────┐   │

│   │  ├── 训练负荷监控 (防止过度训练)                                        │   │
│   │  └── 恢复状态评估 (HRV、睡眠、疲劳度)                                   │   │
│   └───────────────────────────────┬─────────────────────────────────────┘   │
│                                   │                                         │

│   │  ├── 基于数据自动调整计划                                              │   │
│   │  ├── 瓶颈识别与突破建议                                                │   │
│   │  └── 新目标设定 → 回到 1️⃣                                             │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

| 阶段 | 核心能力 | AI 价值 | 用户感知 |
|------|---------|--------|---------|
| **评估** | 多模态诊断 (Vision+IMU+EMG) | 精准识别问题根因 | "终于知道自己哪里不对了" |
| **计划** | 个性化训练生成 | 基于弱点定制，非千篇一律 | "这个计划就是为我设计的" |
| **训练** | 实时反馈纠正 | 每一次练习都有指导 | "像有私教在身边" |
| **追踪** | 进步量化可视化 | 看得见的成长 | "我真的在进步" |
| **优化** | 自适应调整 | 持续学习用户模式 | "它越来越懂我了" |

| **智能鞋垫** | 评估 + 训练 | 重心、足底压力 |

### 4.3 与竞品的本质区别

| 维度 | 竞品 (工具型) | Movement Chain AI (生态型) |
|-----|-------------|--------------------------|
| **定位** | 单次分析工具 | 全程陪伴教练 |
| **用户关系** | 用完即走 | 长期依赖 |
| **服务深度** | 告诉你"哪里错了" | 帮你"改正并进步" |
| **闭环程度** | 评估 only | 评估→计划→训练→追踪→优化 |
| **个性化** | 通用阈值 | 学习用户模式 |
| **目标感** | 无 | 用户设定目标，系统帮助达成 |

!!! success "核心差异化"
    **竞品**: "你的 X-Factor 是 32°，低于职业球员的 45°"

    **我们**: "你的 X-Factor 不足是因为核心激活不够。这周我们重点练习核心激活，每天 3 组平板支撑 + 2 组抗旋转训练。完成后我会重新评估你的进步。"

### 4.4 商业模式演进

| 阶段 | 模式 | 收入来源 | 用户价值 |
|-----|------|---------|---------|
| Phase 1-3 | 硬件 + 订阅 | 传感器销售 + App 会员 | 分析工具 |
| Phase 4 | **服务生态** | 订阅 + API + 内容 + 合作分成 | 目标达成伙伴 |

### 4.5 生态护城河

```text
护城河 = 用户数据积累 + 个性化模型 + 服务闭环粘性
         ↓
用户离开成本极高：历史数据、个性化模型、训练习惯都在平台内
```
