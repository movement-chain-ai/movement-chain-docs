# UI 设计

> **文档目的**: 定义 Movement Chain AI 应用的用户界面设计规范
>
> **目标读者**: UI/UX 设计师、iOS 开发工程师、产品经理

---

## 本节定位

UI 设计规范定义了应用的视觉设计系统、屏幕结构、交互模式和用户流程。

---

## 文档概览

### [屏幕列表](screens.md)

**阅读时间**: ~10 分钟 | **信息架构**

应用的完整屏幕结构：

- **主导航**: 首页、录制、历史、设置
- **功能屏幕**: 评估报告、练习详情、引导界面
- **状态定义**: 空态、加载态、错误态

### [用户流程](user-flows.md)

**阅读时间**: ~15 分钟 | **交互设计**

核心用户场景的完整流程：

- **首次使用**: 注册 → 校准 → 首次评估
- **日常评估**: 录制 → 分析 → 查看报告
- **练习改进**: 报告 → 选择 Drill → 练习 → 验证

### [设计系统](design-system.md)

**阅读时间**: ~10 分钟 | **视觉规范**

视觉设计基础规范：

- **色彩系统**: 品牌色、语义色、状态色
- **字体系统**: 标题、正文、标签层级
- **间距系统**: 8px 网格
- **组件库**: 按钮、卡片、输入框等

### [数据可视化](data-visualization.md)

**阅读时间**: ~10 分钟 | **图表规范**

运动数据的可视化规范：

- **时序图表**: 角速度、角度曲线
- **评分图表**: 雷达图、进度条
- **热力图**: sEMG 激活强度
- **对比视图**: 用户 vs 基准
