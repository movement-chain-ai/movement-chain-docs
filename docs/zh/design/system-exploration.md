## 2. 为什么需要多模态？

### 高尔夫力量链

要真正理解挥杆，需要分析完整的**力量传递链**：

```text
地面 → 双脚 → 腿部 → 髋关节 → 核心 → 肩膀 → 手臂 → 手腕 → 球杆
  ↓      ↓      ↓       ↓       ↓      ↓      ↓       ↓
足底压力  IMU    IMU    视觉    视觉   视觉   EMG    E-Skin

没有单一传感器能捕捉完整力量链 → 必须多模态融合
```

### 竞品都是单一传感器

```text
竞品          视觉    足底    EMG    E-Skin   IMU    评估
───────────────────────────────────────────────────────
SwingMotion    ❌      ❌      ❌      ❌      ✅    单一 IMU
HackMotion     ❌      ❌      ❌      ❌      ✅    单一 IMU
Sportsbox AI   ✅      ❌      ❌      ❌      ❌    单一视觉
K-Motion       ❌      ❌      ❌      ❌      ✅    单一 IMU
BAL.ON         ❌      ✅      ❌      ❌      ❌    单一压力
───────────────────────────────────────────────────────
我们           ✅      ✅      ✅      ✅      ✅    多模态融合
```

## 3. 五种数据源

### 3.1 视觉姿态 (Vision)

**方案**: 手机摄像头 + RTMPose

**能采集**: 17 个身体关键点 (60 FPS)、关节角度、挥杆轨迹

**优点**: 零硬件成本、全身覆盖、无需穿戴

**局限**: 无法测量力量、看不到肌肉发力

**独特价值**: 提供全身空间位置的基准参照系

→ 详细研究: [视觉技术](../components/vision/software.md)

---

### 3.2 足底压力 (Plantar Pressure)

**方案**: 智能鞋垫 (8-32 点压力传感器)

**能采集**: 足底压力分布、重心位置 (CoP)、重心轨迹、左右脚比例

**可用产品**:

| 产品 | 传感器数 | SDK |
|-----|---------|-----|
| BAL.ON Golf | 16+ | ✅ App API |
| Sennotech | 8 | ✅ BLE SDK |
| Nurvv Run | 32 | ✅ SDK |

**独特价值**:

```text
职业选手 vs 业余选手:

    职业: 上杆平滑后移 → 顶点 80% 在右脚 → 下杆快速前移 → 击球时重心已在左脚
    业余: 上杆晃动 → 顶点重心飘移 → 下杆不连贯 → 击球时重心滞后

    视觉只能"看到"动作，压力数据能"量化"重心转移质量
```

→ 详细研究: [压力传感器](../components/pressure-sensors/hardware.md) | [供应商](../components/pressure-sensors/suppliers.md)

---

### 3.3 肌电信号 (EMG)

**方案**: 表面肌电传感器

**能采集**: 肌肉激活时序、发力强度、肌肉协同模式

**可用产品**:

| 产品 | 通道数 | SDK |
|-----|--------|-----|
| DFRobot SEN0240 | 1 | ✅ Arduino |
| OYMotion gForcePro+ | 8 | ✅ SDK |
| Delsys Trigno | 16+ | ✅ SDK |

**独特价值**:

```text
问题: 两人挥杆轨迹相似，但击球距离差 50 码

    高手: 核心先发力 → 力量传递到肩膀 → 最后手臂释放 (延迟释放)
    新手: 手臂先发力 → 核心滞后 (早释放 = 力量损失)

    EMG 能精确检测这个发力顺序差异！
    这是视觉和 IMU 都做不到的。
```

→ 详细研究: [EMG 技术](../components/emg/hardware.md) | [供应商](../components/emg/suppliers.md)

---

### 3.4 柔性传感器 / E-Skin

**方案**: 柔性传感器阵列 (贴片或手套)

**能采集**: 关节弯曲角度、拉伸形变、分布式压力、手指独立运动

**可用产品**:

| 产品 | 类型 | SDK |
|-----|------|-----|
| StretchSense | 电容式 | ✅ |
| BendLabs | 电阻式 | ✅ |
| 能斯达/纽迪瑞 (国产) | 多种 | 需定制 |

**独特价值**: 比 IMU 更贴合身体、可测量 IMU 无法测量的形变

→ 详细研究: [柔性传感器](../components/flexible-sensors/hardware.md) | [供应商](../components/flexible-sensors/suppliers.md)

---

### 3.5 多点 IMU

**方案**: 多个 IMU 模块分布在身体各处

**能采集**: 各部位加速度、角速度、相对旋转角度、高频运动细节

**可用产品**:

| 产品 | 特性 | SDK |
|-----|------|-----|
| WitMotion WT901 | 9轴, BLE | ✅ |
| MetaMotionR | 小型, BLE | ✅ SDK |
| Shimmer3 | 研究级 | ✅ SDK |

**独特价值**: 100-200 Hz 高频采样，视觉在快速运动时模糊，IMU 不会

→ 详细研究: [IMU 技术](../components/imu/hardware.md) | [供应商](../components/imu/suppliers.md)
