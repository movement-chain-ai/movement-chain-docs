# 角色Brief Role Briefs

## 目的

本目录为不同角色提供**针对性的入口文档**，帮助团队成员快速定位自己需要关注的内容，无需阅读整个文档站。

每个Brief聚焦单一角色的工作范围，包含：

- **核心职责** - 该角色需要完成的具体任务
- **必读文档** - 最小必要阅读清单（5-10篇）
- **关键决策** - 影响该角色工作的ADR
- **技术栈** - 使用的工具、框架、语言
- **协作接口** - 与其他角色的交接点

## 使用方法

### 1. 找到你的角色

从上表选择你的角色对应的Brief文档。

### 2. 仅阅读必要内容

**不要尝试阅读整个文档站**。每个Brief已经精选了该角色的最小必要阅读清单（通常5-10篇文档）。

**阅读顺序**：

1. 先读你的Brief文档
2. 按Brief中列出的顺序阅读链接文档
3. 如遇到术语不理解，查阅 [`design/research/biomechanics-glossary.md`](../foundations/biomechanics-glossary.md)

### 3. 参考完整路线图（可选）

如果需要了解项目全貌或跨角色协作细节，可阅读主文档：

- **[系统设计](../architecture/system-design.md)** - 项目级实施路线图（包含所有角色的完整视图）

## Brief文档结构

每个Brief遵循统一的结构：

```text
1. 角色概述
   - 你是谁
   - 你的职责范围

2. 快速开始
   - 第一周要做什么
   - 如何验证环境已就绪

3. 必读文档清单
   - 5-10篇核心文档（带阅读顺序）

4. 关键ADR
   - 影响你工作的技术决策（2-4条）

5. 技术栈和工具
   - 编程语言、框架、开发工具

6. 输入和输出
   - 你依赖什么（上游）
   - 你交付什么（下游）

7. 常见问题FAQ
   - 该角色特有的问题

8. 联系人和资源
   - 相关repo链接
   - 协作对接人
```

## 跨角色协作

虽然每个Brief是独立的，但以下场景需要跨角色沟通：

| 场景 | 涉及角色 | 协作内容 |
|------|----------|----------|
| 传感器数据格式 | 硬件工程师 ↔ 软件工程师 | BLE packet格式、采样率、字节序 |
| ML推理集成 | 软件工程师 ↔ 移动开发者 | ONNX模型输入/输出接口 |
| 反馈规则调优 | 软件工程师 ↔ 高尔夫顾问 | 阈值参数、反馈文案 |
| 硬件调试数据 | 硬件工程师 ↔ 移动开发者 | 实时波形查看、日志导出 |
