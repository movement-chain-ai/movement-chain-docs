## Phase 1: MVP (并行开发策略)

### 目标

> **验证核心价值主张**: 三模态融合 + 双模式闭环 + 实时反馈
>
> **并行开发策略**: Vision、IMU、EMG 三条线同时推进，任一条成功即可发布

### 功能范围

| 功能 | 描述 | 优先级 | 开发线 |
|-----|------|-------|--------|
| **训练场模式** | 固定摄像头 + 可穿戴传感器 | P0 | 主线 |
| **简化实战模式** | 击球后即时语音反馈 | P0 | 主线 |
| **实时骨架** | 3D骨架叠加到视频 | P0 | Vision线 |
| **语音反馈** | 基础问题即时纠正 | P0 | 主线 |
| **挥杆评分** | 综合评分 + 分项指标 | P0 | 主线 |
| **EMG肌肉分析** | 发力时序 + 肌肉激活 | P0 | EMG线 (并行) |
| **历史记录** | 本地挥杆存储和回顾 | P1 | 主线 |
| **简单对比** | 与上次挥杆对比 | P1 | 主线 |

### 技术栈

| 组件 | 选型 | 状态 |
|-----|------|------|
| 姿态估计 | RTMPose + Core ML | 研发中 |
| IMU硬件 | ESP32-S3 + ICM-42688 | 原型完成 |
| iOS App | SwiftUI + AVFoundation | 设计中 |
| 语音合成 | AVSpeechSynthesizer | 现成 |

## Phase 2: 增强版

### 目标

### 新增功能

| 功能 | 描述 | 市场差异化 |
|-----|------|-----------|
| **智能鞋垫集成** | 重心转移分析 | 完整力量链 |
| **触觉反馈** | 振动马达提示 | 增强体验 |
| **3D深度估计** | 单目3D姿态估计 | 技术升级 |
| **训练计划** | AI个性化建议 | 增值服务 |
| **云端同步** | 跨设备数据 | 用户粘性 |
| **防弃用功能** | 见下方详细说明 | 提高留存 |

### 防弃用功能 (Anti-abandonment)

> **背景**: 研究发现高尔夫训练设备普遍在购买后2-4周被用户闲置（"吃灰"）。这些功能旨在提高长期使用率。

| 功能 | 描述 | 优先级 | 是否核心 |
|-----|------|-------|---------|
| **每日挑战** | "今天完成10次挥杆" | P2 | ❌ 非核心 |
| **连续打卡** | 类似Duolingo streak机制 | P2 | ❌ 非核心 |
| **进步徽章** | "本周进步最大"等成就 | P3 | ❌ 非核心 |
| **朋友排行榜** | 社交比较和激励 | P3 | ❌ 非核心 |
| **周报总结** | 每周推送进步报告 | P2 | ❌ 非核心 |

**为什么不是MVP核心功能**:

- MVP核心是验证"挥杆分析+实时反馈"的价值
- 认真的球手会因为核心功能好而持续使用
- 游戏化功能可在有用户数据后再优化
- 避免MVP范围膨胀

## 风险与缓解

### Phase 1 风险

| 风险 | 概率 | 影响 | 缓解措施 |
|-----|------|------|---------|
| 姿态估计精度不足 | 中 | 高 | 备选方案: MediaPipe |
| IMU漂移严重 | 低 | 中 | 增加校准频率 |
| 用户不理解数据 | 中 | 高 | 简化指标 + 语音解释 |
| App审核被拒 | 低 | 中 | 提前了解政策 |

### Phase 2 风险

| 风险 | 概率 | 影响 | 缓解措施 |
|-----|------|------|---------|
| EMG信号噪声大 | 中 | 高 | 干电极优化 + 滤波算法 |
| 实战模式无人用 | 中 | 中 | 用户调研验证 |
| 成本上升过多 | 低 | 中 | 供应链优化 |

### Phase 3 风险

| 风险 | 概率 | 影响 | 缓解措施 |
|-----|------|------|---------|
| AI球童不准确 | 中 | 高 | 大量数据训练 |
| B2B销售困难 | 中 | 中 | 种子客户 + 案例积累 |
| 国际化本地化 | 低 | 中 | 本地团队/合作伙伴 |
