# 产品战略：从市场缺口到设计决策

> **核心命题**: 每一个产品决策都必须对应一个真实的市场痛点

---

## 战略框架：缺口驱动设计

```text
市场痛点          →     设计决策           →      技术实现
─────────────────────────────────────────────────────────────
练习/实战脱节      →     双模式架构          →      模式切换引擎
无实时反馈        →     边缘AI处理          →      端侧推理
数据孤岛         →     统一数据平台         →      跨模式数据同步
信息过载         →     AI教练简化          →      自然语言生成
```text

#### EMG 差异化详解

**为什么 EMG 是杀手级功能？**

| 场景 | 无EMG的问题 | 有EMG的解决方案 |
|-----|-----------|----------------|
| **力量过大** | 只知道"挥太快" | 知道"前臂肌肉过度紧张" |
| **发力顺序错** | 只知道"动作不协调" | 精确显示"髋→肩→臂"时序 |
| **疲劳累积** | 无法检测 | 实时监测肌肉疲劳度 |
| **受伤风险** | 事后才知道 | 提前预警"手腕肌肉异常激活" |

## 二、系统架构设计

### 五模块架构

基于用户痛点和技术能力，我们设计了五模块流水线架构：

```text
┌─────────────────────────────────────────────────────────────────┐
│                    Movement Chain AI 系统架构                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1.评估        2.诊断         3.纠正         4.追踪      5.辅助  │
│  Assessment → Diagnosis →   Correction →   Tracking → Auxiliary │
│                                                                  │
│  ┌─────┐     ┌─────┐        ┌─────┐        ┌─────┐    ┌─────┐  │
│  │多模态│     │链路 │        │个性化│        │数据 │    │AI   │  │
│  │数据 │ →   │问题 │   →    │训练 │   →    │记录 │ →  │球童 │  │
│  │采集 │     │识别 │        │方案 │        │     │    │     │  │
│  └─────┘     └─────┘        └─────┘        └─────┘    └─────┘  │
│     ↓           ↓              ↓              ↓          ↓      │
│  ┌─────┐     ┌─────┐        ┌─────┐        ┌─────┐    ┌─────┐  │
│  │传感器│     │运动链│        │实时 │        │系统 │    │策略 │  │
│  │融合 │     │动力链│        │反馈 │        │洞察 │    │建议 │  │
│  └─────┘     └─────┘        └─────┘        └─────┘    └─────┘  │
│     ↓           ↓              ↓              ↓                 │
│  ┌─────┐     ┌─────┐        ┌─────┐        ┌─────┐             │
│  │评估 │     │诊断 │        │动作 │        │反馈 │             │
│  │输出 │     │输出 │        │回放 │        │闭环 │             │
│  └─────┘     └─────┘        └─────┘        └─────┘             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```text

### 模块详解

#### 模块1：评估 (Assessment)

| 子模块 | 功能 | 输入 | 输出 |
|-------|------|------|------|
| **多模态数据采集** | 收集传感器原始数据 | IMU/EMG/Vision | 原始信号 |
| **传感器融合** | 时间对齐+数据融合 | 多源原始数据 | 统一时序数据 |
| **评估输出** | 生成评估指标 | 融合数据 | 挥杆评分+指标 |

**关键技术**:
- 时间同步: NTP + PTP 协议
- 数据融合: 卡尔曼滤波
- 异常检测: 信号质量监测

#### 模块2：诊断 (Diagnosis)

| 子模块 | 功能 | 输入 | 输出 |
|-------|------|------|------|
| **运动链分析** | 关节序列分析 | 骨骼数据 | 运动链评分 |
| **动力链分析** | 力量传递分析 | EMG+IMU | 发力时序 |
| **交叉验证** | 多源数据验证 | 所有诊断结果 | 置信度评分 |
| **诊断输出** | 问题识别 | 分析结果 | 问题列表+优先级 |

**诊断逻辑示例**:
```text
IF 手腕角度异常 AND 前臂EMG过高:
    → 诊断: "握杆过紧导致手腕僵硬"
    → 置信度: 85%
    → 建议: "放松握杆，想象握鸡蛋"
```text

#### 模块3：纠正 (Correction)

| 子模块 | 功能 | 输入 | 输出 |
|-------|------|------|------|
| **个性化训练** | 生成训练计划 | 诊断结果+历史 | 训练方案 |
| **实时反馈** | 即时纠正 | 当前挥杆 | 语音/视觉/触觉反馈 |
| **动作回放** | 挥杆回顾 | 录制数据 | 3D回放+对比 |

**反馈延迟要求**:
| 模态 | 目标延迟 | 技术实现 |
|-----|---------|---------|
| 触觉 | <100ms | 本地处理 |
| 语音 | <300ms | 端侧TTS |
| 视觉 | <500ms | 异步渲染 |

#### 模块4：追踪 (Tracking)

| 子模块 | 功能 | 输入 | 输出 |
|-------|------|------|------|
| **数据记录** | 持久化存储 | 所有挥杆数据 | 历史数据库 |
| **系统洞察** | 趋势分析 | 历史数据 | 进步曲线+洞察 |
| **反馈闭环** | 方案优化 | 训练效果 | 调整建议 |

**追踪指标**:
- 短期: 每次挥杆评分
- 中期: 周/月进步趋势
- 长期: 整体水平提升

#### 模块5：辅助 (Auxiliary)

| 功能 | 输入 | 输出 | 场景 |
|-----|------|------|------|
| **球杆推荐** | 距离+风向+历史 | 推荐球杆 | 击球前 |
| **策略建议** | 球场布局+能力 | 攻略路线 | 开球前 |
| **心理辅导** | 连续失误检测 | 鼓励话语 | 状态低迷时 |
