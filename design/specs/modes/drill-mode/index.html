    练习模式¶

    文档目的: 研究基于数据的个性化练习推荐系统

    状态: 研究提案 (待讨论)

    最后更新: 2025 年 1 月 5 日

    1. 概述¶

    根据评估结果生成有针对性的改进训练动作建议:

    竞品: "你的 X-Factor 只有 35°"  →  用户: 我知道了，但怎么改？

我们:  "你的 X-Factor 只有 35°，因为核心肌群激活延迟了 150ms。
       推荐: Pelvis-First Drill + Core Activation (3组×15次)"
       →  用户: 我知道为什么，也知道怎么练

    2. 类似产品¶

    核心痛点: "练习场 vs 球场差距" (85% 用户提及) — 本质是缺乏针对性训练指导。

    竞品缺陷: 所有竞品都是"猜测"根因，只有我们能用 EMG 证明。

    竞品 能做 不能做
    Sparrow Up 50+ Drills 推荐 不知道为什么这个 Drill 适合你. 具体分析见 5. 竞品分析与差异化
    OnForm/Sportsbox 角度、运动链 不知道哪块肌肉没发力

    机会: 基于 imu + vision + EMG 诊断推荐针对性训练，形成"诊断 → 根因 → 训练"完整闭环。

    3. 架构集成方案¶

    3.1 Stage 6 扩展设计¶

    根据 系统设计文档，训练推荐集成到 Stage 6 AI 反馈层：

    ┌─────────────────────────────────────────────────────────────────────────┐
│                    扩展后的 Stage 6 数据流                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  输入:                                                                  │
│  ├── triggered_rules: ["ARMS_BEFORE_CORE", "LOW_X_FACTOR"]             │
│  ├── metrics: {x_factor: 35°, core_activation: 38%, ...}               │
│  └── user_history: {skill_level: "intermediate", ...}                  │
│                                                                         │
│  处理:                                                                  │
│  ├── 1. Rule Engine 确定诊断结果                                        │
│  ├── 2. Training DB 查询匹配的训练动作                                   │
│  │      query: diagnosis="ARMS_BEFORE_CORE" AND target="core"          │
│  │      result: [{drill: "Pelvis-First Drill", video: "...", ...}]     │
│  ├── 3. Enhanced Prompt 构建                                            │
│  │      + 诊断结果 + EMG 根因解释 + 推荐训练动作                         │
│  └── 4. LLM 生成自然语言反馈                                            │
│                                                                         │
│  输出:                                                                  │
│  ├── feedback_text: "你的核心激活延迟了..."                             │
│  └── recommended_drills: [{name, video_url, sets, reps, duration}]     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

    3.2 Function Calling 架构¶

    AI Agent 工具集:
├── diagnose_swing(metrics) → diagnosis[]
├── explain_root_cause(diagnosis, emg_data) → explanation
├── query_training_db(diagnosis, user_profile) → drills[]
├── generate_training_plan(drills, user_schedule) → plan
└── track_progress(user_id, drill_id) → progress_stats

    4. 训练动作数据库设计¶

    4.1 专业术语 (Golf Training Terminology)¶

    英文术语 中文 定义
    Golf Drills 高尔夫练习 针对特定挥杆问题的重复练习
    Corrective Exercises 矫正练习 修复身体限制或代偿模式
    Mobility Training 灵活性训练 提高关节活动范围
    Motor Pattern Drill 动作模式练习 重建正确的神经肌肉模式

    4.2 诊断 → 训练映射表¶

    根据 生物力学基础 已有的映射：

    MVP 诊断规则 EMG 根因 推荐训练 类型
    ARMS_BEFORE_CORE 前臂先于核心激活 Pelvis-First Drill Motor Pattern
    ARMS_BEFORE_CORE 核心激活不足 (<40%) Dead Bug, Pallof Press Strength
    FALSE_COIL 肩转但核心不激活 Loaded Rotation Drill Motor Pattern
    LOW_X_FACTOR 胸椎灵活性不足 Thoracic Spine Rotation Mobility
    LOW_X_FACTOR 髋部限制 90/90 Hip Stretch Mobility
    FAST_TEMPO 过渡太急 Pause-at-Top Drill Motor Pattern
    SLOW_TEMPO 下杆减速 Step Drill Motor Pattern
    EARLY_RELEASE 手腕提前释放 Lag Preservation Drill Motor Pattern

    5. 竞品分析与差异化¶

    5.1 Sparrow Up 功能拆解¶

    直接竞争对手

    Sparrow Up 是唯一提供"训练推荐"功能的高尔夫分析 App

    信息 内容
    官网 sparrowup.com/golf
    定价 $99.99/年
    Drills 50+ 训练动作
    EMG 数据 ❌ 无

    Sparrow Up 工作流:

    视频分析 → 问题检测 → Drill 推荐
   │           │           │
   ▼           ▼           ▼
 能看到      能判断      能推荐
 关节角度    早释/过顶    通用 Drill
   ❌          ❌          ❌
 看不到肌肉  不知为什么   无个性化

    5.2 对比矩阵¶

    维度 Sparrow Up Movement Chain 优势方
    诊断能力 视觉派生 视觉 + IMU + EMG 🏆 我们
    根因解释 ❌ 猜测 ✅ EMG 证明 🏆 我们
    Drills 数量 50+ 20+ (MVP) Sparrow Up
    个性化程度 低 (通用匹配) 高 (EMG 驱动) 🏆 我们
    进度验证 视觉对比 EMG 数据对比 🏆 我们

    5.3 完整竞品矩阵¶

    功能 OnForm Sportsbox Sparrow Up GolfFix 我们
    视觉分析 ✅ ✅ ✅ ✅ ✅
    IMU 数据 ❌ ❌ ❌ ❌ ✅
    EMG 数据 ❌ ❌ ❌ ❌ ✅
    训练推荐 ❌ ❌ ✅ ✅ ✅
    EMG 驱动推荐 ❌ ❌ ❌ ❌ ✅

    5.4 技术护城河¶

    ┌─────────────────────────────────────────────────────────────────────────┐
│  竞品想复制我们的训练推荐系统，需要:                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 硬件投资 (6-12 个月)                                                │
│     └── EMG 传感器开发 + IMU 集成 + 供应链建立                          │
│                                                                         │
│  2. 算法研发 (专业知识)                                                 │
│     └── EMG 信号处理 + 诊断规则 + 时间同步                              │
│                                                                         │
│  3. 数据积累 (需要时间)                                                 │
│     └── EMG + 挥杆数据配对 + 训练效果验证                               │
│                                                                         │
│  ⏱️ 估计追赶时间: 18-24 个月                                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
    优势 可复制性 领先时间
    EMG 硬件集成 难 18+ 月
    诊断 → 根因解释 难 12+ 月
    训练效果 EMG 验证 不可能 ∞
