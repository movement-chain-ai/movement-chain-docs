    用户流程¶

    文档目的: 定义核心用户场景的完整交互流程

    目标读者: UI/UX 设计师、产品经理、前端开发工程师

    设计主题: 金稻绿野 (Golden Harvest)

    1. 核心流程概览¶

    ┌─────────────────────────────────────────────────────────────────────────────┐
│                           核心用户流程                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   用户认证                                                                   │
│   ────────                                                                  │
│   启动 App → 登录/注册 → 首页                                                │
│                                                                             │
│   评估模式                                                                   │
│   ────────                                                                  │
│   选择评估 → 传感器检查 → 选择类型 → 三步评估 → 处理中 → 报告                  │
│                                                                             │
│   练习模式                                                                   │
│   ────────                                                                  │
│   选择练习 → 练习库/训练计划 → 传感器检查 → 执行练习 → 完成                    │
│                                                                             │
│   引导模式                                                                   │
│   ────────                                                                  │
│   选择引导 → 传感器检查 → 站姿/慢动作引导 → 实时反馈 → 完成                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

    2. 用户认证流程¶

    2.1 流程图¶

    flowchart TD
    A[启动 App] --> B{已认证?}
    B -->|否| C[AuthPage]
    B -->|是| M[HomePage]

    C --> D{登录/注册切换}
    D -->|登录| E[输入邮箱密码]
    D -->|注册| F[输入姓名/邮箱/密码]

    E --> G[点击登录]
    F --> H[点击注册]

    G --> I{验证通过?}
    H --> J{注册成功?}

    I -->|是| M
    I -->|否| E
    J -->|是| M
    J -->|否| F

    C --> K[社交登录]
    K --> L[Google/Apple/Twitter]
    L --> M

    2.2 页面状态¶

    状态 文件 说明
    登录表单 AuthPage.tsx 默认显示，包含邮箱、密码输入
    注册表单 AuthPage.tsx 切换后显示，增加姓名输入
    社交登录 AuthPage.tsx Google / Apple / Twitter 快捷登录

    2.3 界面元素¶

    // 认证表单字段
interface AuthForm {
  isLogin: boolean;      // 登录/注册切换
  email: string;         // 邮箱
  password: string;      // 密码 (最少6位)
  name?: string;         // 姓名 (仅注册)
}

    3. 评估模式流程¶

    3.1 流程图¶

    flowchart TD
    A[首页/模式页] --> B[点击评估模式]
    B --> C[AssessmentMode - ready]

    C --> D[显示传感器状态]
    D --> E{传感器正常?}
    E -->|是| F[显示就绪提示]
    E -->|否| G[显示警告/可跳过]

    F --> H[点击继续]
    G --> H

    H --> I[intro - 选择评估类型]

    I --> J{选择类型}
    J -->|单项评估| K[选择 Step 1/2/3]
    J -->|完整评估| L[开始三段式]

    K --> M[执行单步]
    M --> N[步骤完成]
    N --> O{继续/生成报告?}
    O -->|继续| K
    O -->|生成| P[processing]

    L --> Q[Step 1: 站姿检查]
    Q --> R[Step 2: 慢动作检查]
    R --> S[Step 3: 全速检查]
    S --> P

    P --> T[report - 评估报告]
    T --> U{用户操作}
    U -->|查看阶段详情| V[swing-phase-detail]
    U -->|进入练习| W[DrillMode]
    U -->|重新评估| I
    U -->|返回首页| X[HomePage]

    V --> T

    3.2 评估步骤状态¶

    type AssessmentStep =
  | 'intro'           // 选择评估类型
  | 'ready'           // 传感器检查
  | 'step1'           // 站姿检查进行中
  | 'step1-complete'  // 站姿检查完成
  | 'step2'           // 慢动作检查进行中
  | 'step2-complete'  // 慢动作检查完成
  | 'step3'           // 全速检查进行中
  | 'step3-complete'  // 全速检查完成
  | 'processing'      // 生成报告中
  | 'report'          // 显示报告
  | 'swing-phase-detail';  // 挥杆阶段详情

    3.4 评估报告数据结构¶

    interface AssessmentData {
  score: number;          // 综合评分 (0-100)
  date: string;           // 评估日期
  problems: Problem[];    // 发现的问题
  strengths: string[];    // 优势项
}

interface Problem {
  id: string;
  title: string;          // 问题标题
  description: string;    // 问题描述
  priority: 'high' | 'medium' | 'low';  // 优先级
}

    4. 练习模式流程¶

    4.1 流程图¶

    flowchart TD
    A[首页/模式页] --> B[点击练习模式]
    B --> C[DrillMode]

    C --> D{Tab 选择}
    D -->|练习计划| E[TrainingPlan]
    D -->|练习库| F[练习分类列表]

    E --> G{选择计划}
    G -->|查看详情| H[PlanDetail]
    G -->|开始训练| I[PlanExecutor - 传感器检查]

    H --> J{操作}
    J -->|开始训练| I
    J -->|返回| E

    I --> K{传感器正常?}
    K -->|是| L[AllDrillsMode]
    K -->|否| M[警告提示]
    M --> L

    L --> N[执行练习]
    N --> O[练习完成]
    O --> C

    F --> P[选择分类]
    P --> Q[查看 Drill 列表]
    Q --> R[点击 Drill]
    R --> S[DrillDetail]

    S --> T{操作}
    T -->|开始练习| U[GuidedPractice]
    T -->|返回| Q

    U --> V[完成练习]
    V --> C

    4.3 练习数据结构¶

    interface Drill {
  id: string;                // 唯一标识 (如 PFD001)
  name: string;              // 英文名称
  nameZh: string;            // 中文名称
  category: DrillCategory;   // 分类
  targetProblem: string;     // 针对问题
  difficulty: 'beginner' | 'intermediate' | 'advanced';
  sets: number;              // 组数
  reps: number;              // 次数
  duration?: string;         // 时长描述
  description: string;       // 练习说明
}

// 难度显示
const difficultyMap = {
  beginner: { label: '初级', color: 'var(--color-success)' },
  intermediate: { label: '中级', color: 'var(--color-priority-medium)' },
  advanced: { label: '高级', color: 'var(--color-priority-high)' }
};

    5. 引导模式流程¶

    5.1 流程图¶

    flowchart TD
    A[首页/模式页] --> B[点击引导模式]
    B --> C[GuideMode - ready]

    C --> D[显示传感器状态]
    D --> E[点击继续]

    E --> F[intro - 选择引导类型]

    F --> G{选择类型}
    G -->|站姿引导| H[GuidedPractice - 站姿]
    G -->|慢动作引导| I[GuidedPractice - 慢动作]

    K --> L[完成]

    I --> M[阶段语音提示]
    M --> N[完成]

    J --> O[Phase 1: 站姿检查]
    O --> P[Phase 2: 挥杆练习]
    P --> Q[完成]

    L --> F
    N --> F
    Q --> F

    5.2 引导类型状态¶

    type GuideType =
  | 'intro'       // 选择引导类型

  | 'practice'    // 引导练习中
  | 'setup'       // 站姿引导
  | 'slowmotion'  // 慢动作引导
  | 'combined'    // 综合引导
  | 'allDrills';  // 全部练习模式

type CombinedPhase = 'stance' | 'slowmotion';

    6. 传感器管理流程¶

    6.1 传感器状态¶

    interface Sensor {
  id: string;
  name: string;
